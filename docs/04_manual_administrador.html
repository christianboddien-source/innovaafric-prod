1. Acceso al Panel de Administración

1.1 Credenciales y Niveles de Acceso

El panel de administración está restringido a usuarios con nivel 2 o superior. El acceso se realiza con las mismas credenciales del ecosistema INNOVAAFRIC pero el nivel debe estar configurado por un superadmin.

NivelRolAcceso
Nivel 2StaffVer usuarios, ver transacciones, dashboard básico, tickets de soporte
Nivel 3AdminTodo de Nivel 2 + gestión KYC, baneos, notificaciones, compliance, API keys lectura
Nivel 4SuperadminAcceso completo: crear/revocar API keys, nóminas, configuración del sistema, gestión de roles
Seguridad
Nunca compartas credenciales de administrador. Cada acción realizada queda registrada con el ID del administrador y la marca de tiempo. Los accesos de Nivel 4 deben revisarse trimestralmente.

3. Gestión de Usuarios

3.1 Buscar y Ver Perfil

  1. Ve a Usuarios en el menú lateral.
  2. Usa los filtros: búsqueda por nombre/email, país, rol, estado KYC.
  3. Haz clic en un usuario para ver su perfil completo.
  4. El perfil muestra: datos personales, KYC, wallets, historial de transacciones, tickets y notas admin.

3.2 Banear / Desbanear Usuario

  1. Abre el perfil del usuario.
  2. Haz clic en "Banear".
  3. Introduce el motivo del baneo y la duración (en días o indefinido).
  4. Confirma. El usuario quedará bloqueado inmediatamente.
  5. Para desbanear: abre el perfil del usuario baneado y haz clic en "Levantar baneo".
Registro de Auditoría
Cada baneo/desbaneo queda registrado con: admin responsable, motivo, fecha y duración. Este historial es inmutable.

3.3 Aprobar KYC Manualmente

  1. Ve a KYC en el menú o filtra usuarios por kyc_status = pending.
  2. Revisa el documento subido (se abre desde Supabase Storage).
  3. Selecciona "Aprobar" o "Rechazar".
  4. En caso de rechazo, escribe el motivo que se enviará al usuario por notificación.

6. Notificaciones

6.1 Notificación Masiva

  1. Ve a Notificaciones → Nueva notificación masiva.
  2. Escribe el título y el cuerpo del mensaje.
  3. Selecciona el tipo: info, warning, system, promotion.
  4. Filtra el destino: todos los usuarios, por país, por rol, o por estado KYC.
  5. Previsualiza y confirma el envío.
Endpoint usado
POST /v1/admin/notifications/mass — Las notificaciones se crean en la tabla notifications de Supabase.

6.2 Notificación Individual

Desde el perfil de un usuario específico, usa "Enviar notificación" para enviar un mensaje personalizado.

7. Gestión de API Keys
Solo Nivel 4 (Superadmin)
La creación y revocación de API keys requiere nivel 4. Las claves completas solo se muestran en el momento de creación. Guárdalas de forma segura.

7.1 Crear API Key

  1. Ve a API Keys → Nueva clave.
  2. Introduce el nombre del partner.
  3. Selecciona el entorno: production o sandbox.
  4. Define los permisos: read:users, read:transactions, write:orders, etc.
  5. Establece el límite de peticiones por minuto.
  6. Haz clic en "Crear". Copia y guarda la clave completa — solo se muestra una vez.

7.2 Revocar API Key

Desde la lista de API keys, cambia el estado a revoked. Las peticiones con esa clave devolverán 401 inmediatamente.

8. Compliance y AML

8.1 Informes CTR (Currency Transaction Report)

Los CTR se generan automáticamente cuando una transacción supera los umbrales regulatorios, o manualmente por un admin. Se accede desde Compliance → CTR Reports.

8.2 Casos de Compliance

EstadoDescripciónAcción

under_reviewEn proceso de investigaciónDocumentar hallazgos
escalatedEscalado a autoridades regulatoriasPreparar SAR (Suspicious Activity Report)

8.3 SAR — Suspicious Activity Report

Desde un caso escalado, utiliza "Generar SAR" para crear el informe de actividad sospechosa. El documento se exporta en PDF y queda archivado en el sistema.

11. Tickets de Soporte
  1. Ve a Soporte → Tickets.
  2. Filtra por estado (open, in_progress, resolved) y prioridad (low, medium, high, urgent).
  3. Abre el ticket, lee el historial de mensajes del usuario.
  4. Responde directamente desde el panel — el usuario recibe notificación.
  5. Cambia el estado del ticket según corresponda.
  6. Si requiere escalado técnico, asigna a un staff específico.
12. Nóminas (Payroll)
Solo Nivel 4
La gestión de nóminas requiere nivel 4 (Superadmin).

12.1 Crear una Ejecución de Nómina

  1. Ve a Nóminas → Nueva ejecución.
  2. Introduce el período de nómina (ej. Junio 2026).
  3. Añade ítems de nómina: empleado, importe, moneda, tipo (salario/bonus/comisión).
  4. Revisa el total antes de aprobar.
  5. Haz clic en "Aprobar ejecución". Los pagos se procesan automáticamente.
13. Sistema de Fallback de 3 Niveles

El Admin Panel utiliza un sistema de 3 niveles de fallback para garantizar disponibilidad máxima incluso cuando algún servicio no responde.

1

Railway API (Principal)

Datos en tiempo real desde la API REST de Railway. Todas las operaciones de escritura se realizan aquí. Si Railway responde en <5s, se usa este nivel.

2

Supabase (Secundario)

Si Railway no responde, el panel lee directamente de Supabase usando el SDK de JavaScript. Permite ver usuarios y transacciones pero con funcionalidad de escritura limitada.

3

localStorage Cache (Último recurso)

Si tanto Railway como Supabase no responden, el panel muestra los últimos datos cacheados en localStorage del navegador. Solo lectura. Los datos pueden tener hasta 24h de antigüedad.

Indicador de Estado del Sistema
El panel muestra en la esquina superior derecha qué nivel de fallback está activo: verde (Railway), naranja (Supabase) o rojo (cache local).
